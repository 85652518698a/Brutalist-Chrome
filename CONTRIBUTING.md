# Contributing to Brutalist Chrome

All contributions are welcome. Keep it brutal — function first.

## Rules
1. No unnecessary dependencies
2. No rounded corners (seriously)
3. Every feature must justify its file size
4. Comment your code
5. Test in Chrome before PRing

## How to Contribute
1. Fork the repo
2. Create a branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Test with `node scripts/build.js`
5. Open a Pull Request

## Code Style
- 2-space indentation
- camelCase for JS, kebab-case for CSS
- Uppercase for constants
- Monospace font for all UI labels
